## 1. Scop

Materiale gata de folosit ca PM-ul să ruleze **3–5 interviuri de validare** cu agenții imobiliare din Chișinău, cât timp dezvoltarea e în M1.S3. Scopul: să confirmăm problema și dispoziția de adopție **înainte** de a investi mai mult în M1.S4-S6, și să lăsăm feedback-ul să re-prioritizeze roadmap-ul.

**Principiu:** validare, nu vânzare. Ascultăm. Promitem doar ce există în demo + roadmap-ul aprobat.

## 2. Conținut kit (5 fișiere)

| # | Fișier | Ce este | Când se folosește |
|---|---|---|---|
| 1 | `ONE_PAGER.md` | Rezumat 1 pagină printabil: problema agentului RM → ce face REVYX → ce NU e (CRM) → cerere pilot 30 zile | Trimis/înmânat **înainte** de întâlnire |
| 2 | `INTERVIEW_GUIDE.md` | 12 întrebări deschise + script demo 10 min (4 fluxuri) | **În timpul** întâlnirii |
| 3 | `FEEDBACK_FORM.md` | 5 scale 1–5 (mapate pe piloni) + 3 întrebări deschise | **După** demo |
| 4 | `SUCCESS_CRITERIA.md` | Praguri de succes + tabel de centralizare + catalog obiecții | La **sinteză** (după toate interviurile) |
| — | `README.md` | Acest index + fluxul de utilizare | Referință |

## 3. Fluxul unui interviu (≤ 45 min)

```
[înainte]  Trimite ONE_PAGER → programează 45 min
   │
[0–30 min] INTERVIEW_GUIDE §2 — 12 întrebări deschise (ascultă 80%, nu vinde)
   │
[30–40]    INTERVIEW_GUIDE §3 — demo live 10 min (calificare → potriviri → kanban → cabinet)
   │
[40–45]    FEEDBACK_FORM — completat împreună (5 scale + 3 deschise + decizii pilot)
   │
[după]     Transferă în SUCCESS_CRITERIA §3 (tabel centralizare) + 1 citat verbatim
```

După 3–5 interviuri → evaluează pragurile din `SUCCESS_CRITERIA.md` §1 → predă către **VAL-02** (sinteză + re-prioritizare M1.S4-S6).

## 4. Reguli de conținut respectate
- **Fără promisiuni de features inexistente** — doar ce e în demo (`apps/web-preview/`, 26 ecrane) + roadmap aprobat.
- **Formulele de scoring rămân ascunse în UI/materiale** — vorbim în beneficii (prioritate, direcție, stare), nu acronime (LS/PS/NBA/DHI).
- **Ton brand** — profesional · precis · executiv (`brand-configs/revyx.md` §7).
- Scalele feedback sunt **mapate 1:1 pe piloni** (BRD §5) ca răspunsurile să se traducă direct în priorități M1.S4-S6.
